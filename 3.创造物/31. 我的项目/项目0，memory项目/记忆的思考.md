### **MemOS**
**MemOS 的技术路线：Memory = 可训练的神经结构（Tensor Memory）**

  

与其他所有系统不同，MemOS 的哲学是：

  

**不仅存数据，连记忆本身也应该是可学习的结构。**

  

核心组件包括：

- MemCube（张量化记忆单元）统一管理明文、激活、参数三种记忆
- 系统架构层：接口层、操作层、基础设施层三层架构，将记忆"资源化"，实现了专业化分工与统一调度

  

**现实问题：**

- 学习曲线陡峭，集成需要较多定制开发
- 系统较重，不适合轻量级场景

  

**MemOS 的定位**

- 具有未来记忆技术的研究气质
- 对学术与前沿记忆模型有探索意义

### **MemU**
MemU 最独特的地方在于它把**记忆组织成树（Memory Tree）。**

  

结构特点：

  

- 层级化存储架构：通过根、枝、叶三级节点，实现记忆的自动分类存储。
- 动态扩展机制：新记忆基于语义，自动归入对应分支，实现自我生长。
- 权重检索：系统按使用频率等因素计算权重，并以此决定检索优先级。

  

这让它在：

- 用户画像
- 档案管理
- 教育记录
- 结构化任务

  

表现极好。

但它缺乏：

- 开放域推理
- 噪声鲁棒性
- 灵活度

### **EverMemOS（2025-09 → 2025-11）——第二代 Memory OS：AI 的长期"大脑"诞生**

verMemOS创新性地构建了仿生四层记忆架构，体现了人脑记忆机制的工程化模拟

  

**①分类记忆提取**

  

根据情景、语义、画像等不同记忆类型，进行提取和分层分区存储，既保持对立有可有效关联

  

**② MemCell原子化存储**

  

每条记忆带时间、来源、标签、关系等，类似人脑中的记忆单元

  

**③ 事件边界**

  

不按会话或token强行切分，而是按主题关联跨会话定义事件，形成可理解的记忆。

  

**④ 多重召回**

  

对简单请求作快速召回，对复杂请求进行多跳推理形成深度召回，类似人脑的前额叶和海马体的协作。
