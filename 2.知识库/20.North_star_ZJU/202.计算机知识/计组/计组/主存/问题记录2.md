1. **双端口存储器的冲突**：
    
    - **读写冲突**：当两个端口同时访问**同一个存储单元**（地址相同），且至少有一个端口执行写操作时，会发生冲突。
    - **无冲突**：如果两个端口访问不同地址，或同时读同一地址，则不会冲突。

| **对比维度**     | **主存块号**                             | **物理地址**                      |
| ------------ | ------------------------------------ | ----------------------------- |
| **定义**       | 主存中数据块的索引编号，标识主存块的唯一性                | 存储单元的实际硬件地址，用于直接访问内存数据        |
| **组成结构**     | 由高位标记和低位组号构成（取决于映射方式）                | 包含主存块号（或标记、组号）和块内偏移量          |
| **用途**       | 用于Cache映射和主存数据定位                     | 直接访问内存中的具体存储单元                |
| **生成方式**     | 由主存地址高位计算或地址转换得到                     | 由主存块号与块内偏移量拼接而成               |
| **容量关系**     | 主存块数 = 主存总容量 / 块大小                   | 物理地址空间 = 主存块数 × 块大小 + 偏移范围    |
| **示例（组相联）**  | 主存块号 = 标记 + 组号（如4102 → Tag=8, Set=6） | 物理地址 = 标记（8）\| 组号（6）\| 偏移（6位） |
| **与Cache关系** | 决定数据在Cache中的映射位置（组号）                 | 通过地址划分定位Cache中的具体存储单元         |
| **数据交换单位**   | 主存与Cache交换数据的基本单位（如64B）              | 单个字节或字的访问单位                   |
采用组相联时，主存块号！主存块号%组号就能算这个块号的主存要被映射到哪个地方。

影响 Cache命中率的因素有Cache的容量、Cache结构、每次与主存交换信息的单位量和不同的映射方式。
