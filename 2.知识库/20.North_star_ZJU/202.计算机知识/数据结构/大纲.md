# Bedrock
- **第一阶段：线性结构 (已完成80%)**
    
    - **内容：** 顺序表、链表、栈、队列。
        
    - **状态：** 顺序表、链表已攻克。栈与队列已开启。
        
    - **预计剩余时间：** **1周**。我们将完成栈与队列的核心应用题，并进行一次全面的线性结构总复盘。
        
- **第二阶段：非线性结构 (核心与难点)**
    
    - **Part A: 树 (Tree)** - **预计时间：4-5周**
        
        - **基础：** 二叉树的定义、性质、存储结构。
            
        - **核心战技：** 二叉树的**遍历**（前序、中序、后序、层序）的**递归与迭代**实现。这是绝对的重中之重。
            
        - **高级堡垒：** 二叉搜索树(BST)、平衡二叉树(AVL)、堆(Heap)。
            
        - **应用：** 哈夫曼树、并查集。
            
    - **Part B: 图 (Graph)** - **预计时间：3-4周**
        
        - **基础：** 图的定义、术语、邻接矩阵与邻接表存储。
            
        - **核心战技：** 图的**遍历**（深度优先DFS、广度优先BFS）。
            
        - **高级堡垒：** 最小生成树（Prim, Kruskal）、最短路径（Dijkstra, Floyd）、拓扑排序、关键路径。
            
- **第三阶段：查找与排序 (算法的集大成)**
    
    - **Part A: 查找 (Searching)** - **预计时间：2周**
        
        - **基础：** 顺序查找、二分查找。
            
        - **核心：** 哈希表（冲突处理方法是考研重点）。
            
    - **Part B: 排序 (Sorting)** - **预计时间：3-4周**
        
        - **基础排序：** 插入、选择、冒泡。
            
        - **核心排序 (必考)：** **希尔排序、堆排序、归并排序、快速排序**。您需要掌握每一种的算法思想、代码实现、时间/空间复杂度（最好/最坏/平均）、以及**稳定性**。
            
        - **特殊排序：** 基数排序。
            
- **第四阶段：总复盘与模拟 (查漏补缺)**
    
    - **预计时间：1周**
        
    - 我们将进行一次跨章节的、模拟408选择题的“理论速射”，并挑战1-2道需要组合多种数据结构才能解决的“综合大题”，检验您的知识体系是否已经“融会贯通”。

# 考试

#### **第一战场：单项选择题 (约占1/3分数，~15分)**

这是对您**“理论基础”**和**“概念精度”**的全面“扫射”。每道题看似简单，但往往暗藏陷阱。

**主要“火力点” (高频考点):**

1. **基本概念与术语：**
    
    - **考法：** 给你一段描述，问它说的是哪种逻辑结构/存储结构？（例如：“数据元素之间存在一对一关系” -> 线性结构）
        
    - **例子：** 数据结构、逻辑结构、物理结构、算法的定义和特性（有穷性、确定性等）。
        
2. **时间/空间复杂度分析：**
    
    - **考法：** 给你一段简短的、甚至有嵌套循环或递归的代码，让你快速计算出其时间复杂度。或者，问你某个经典算法（如二分查找）的时间复杂度。
        
    - **例子：** for(i=1; i<=n; i*=2) 的时间复杂度是多少？（O(log n)）
        
3. **各种数据结构的“特性”对比：**
    
    - **考法：** 这是**绝对的重点和难点**！题目会设置一个场景，问你“在这种场景下，使用哪种数据结构最高效/最合适？”
        
    - **例子：**
        
        - “需要在大量数据中，频繁地进行插入和删除操作，但很少进行随机访问，应优先选用？” -> **链表** (而非顺序表)。
            
        - “为了实现一个具有‘后进先出’特性的功能，应选用？” -> **栈**。
            
        - “哪种排序算法是‘不稳定’的？” -> **选择排序、快速排序、堆排序**。
            
        - “哪种结构，其查找效率与数据量大小无关？” -> **哈希表（在无冲突的理想情况下）**。
            
4. **特定数据结构的操作过程与结果：**
    
    - **考法：** 给你一个初始状态和一系列操作，问你最终的状态或结果是什么。
        
    - **例子：**
        
        - 一个序列 [a, b, c, d] 依次入栈，可能的出栈序列有哪些？
            
        - 对一棵二叉树，进行“中序遍历”得到的结果是什么？
            
        - 将一组数，依次插入一棵“二叉排序树”，最终树的形态是怎样的？树的高度是多少？
            
        - 对一个图，进行“深度优先遍历(DFS)”或“广度优先遍历(BFS)”，得到的顶点序列是什么？
            

#### **第二战场：综合应用题 (算法设计题) (约占2/3分数，~30分)**

这是**“决胜局”**，是对您**“将理论应用于实践”的编码能力**的终极考验。通常有2道大题，每道题10-15分。

**主要“出招方式” (题型):**

1. **“魔改”经典算法：**
    
    - **考法：** 题目背景往往是一个我们熟悉的数据结构（顺序表、链表、树），但要求你实现一个**非标准**的功能。
        
    - **例子：**
        
        - 我们刚做的**“反转链表的一部分”**，就是典型的例子。
            
        - “请设计一个算法，找到一个单链表的**倒数第k个**节点。” (快慢指针的又一应用)
            
        - “请将一个顺序表中的奇数，全部移动到偶数的前面。” (双指针的应用)
            
        - “判断一棵二叉树，是否是‘平衡二叉树’。” (树的递归遍历)
            
2. **“组合”多种数据结构：**
    
    - **考法：** 问题需要你结合多种数据结构的特性来解决。
        
    - **例子：** “设计一个算法，找到二叉树中两个指定节点的最-近公共祖先(LCA)。” (可能需要递归+后序遍历的思想)。我们之前做的“和为K的子数组”，也是“顺序表+哈希表”的组合。
        
3. **设计全新的“伪”数据结构：**
    
    - **考法：** 题目会描述一个具有特定操作限制的、新的数据结构模型，要求你用基本的数据结构（数组、链表）来实现它，并分析其效率。
        
    - **例子：** “请用两个栈，来实现一个队列。” 或者 “请设计一个特殊的栈，除了常规push/pop，还能在O(1)时间内，返回当前栈中的最小值。”
        

**评分标准：**

- **算法思想 (最重要)：** 你的解法是否正确、高效？思路是否清晰？（占分比最高）
    
- **代码实现：** 使用**类C语言的伪代码**写出完整、正确的算法。关键步骤需要有注释。
    
- **复杂度分析：** 必须明确指出你的算法的时间复杂度和空间复杂度。
    
- **边界处理：** 是否考虑了空链表、空树等特殊情况？
