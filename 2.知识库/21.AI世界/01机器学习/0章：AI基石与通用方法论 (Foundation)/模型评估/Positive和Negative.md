### **什么“Positive”通常代表“需要被检测出来的少数”？**

这里的 Positive/Negative 标签，并不是生活中的“好/坏”，而是来自于统计学和假设检验的术语。它的含义更接近于：

- **Positive (阳性)**：代表“**事件发生了**”、“**信号被检测到了**”或者“**是我们重点关注和寻找的目标**”。
    
- **Negative (阴性)**：代表“**事件未发生**”、“**正常状态**”或者“**背景**”。
    

我们来看几个经典场景，你就能理解这个惯例了：

|   |   |   |   |
|---|---|---|---|
|场景|**Positive (关注的目标)**|**Negative (正常/背景)**|我们不希望发生什么？|
|**疾病诊断**|患病 (呈阳性)|健康 (呈阴性)|**FN (漏诊)**：把病人当健康人|
|**垃圾邮件**|是垃圾邮件|是正常邮件|**FP (误杀)**：把正常邮件当垃圾|
|**金融欺诈**|是欺诈交易|是正常交易|**FN (漏报)**：把欺诈交易当正常|
|**系统入侵**|检测到入侵|系统正常|**FN (漏报)**：没发现黑客入侵|
|**次品检测**|是次品|是合格品|**FN (漏报)**：把次品当合格品卖了|

**结论：**  
在绝大多数二分类问题中，“Positive”类通常是：

1. **少数类 (Minority Class)**：病人的数量远少于健康人。
    
2. **我们更感兴趣、需要采取行动的类**：发现了疾病就要治疗，发现了欺诈就要阻止。
    
3. **预测错误的代价通常不均衡的类**：漏诊病人的代价，远大于把健康人叫来复查的代价。
    

所以，请记住这个约定俗成的惯例：**Positive 通常不代表“好”，而是代表“需要被揪出来的那个目标”**。你之前的分析，只要把这个“好/坏”的标签反过来，就完全正确了。
