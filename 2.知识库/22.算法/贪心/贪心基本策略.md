# 区间题
建立贪心直觉：

- **无重叠区间**：我保留了结束最早的区间，这样我就能**更早地开始考虑下一个不重叠的区间**，从而更有可能保留更多。
    
- **引爆气球**：我在结束最早的那个区间的终点射箭，这一箭能覆盖所有与它重叠的区间。因为这个终点位置非常靠前，所以它**最不可能影响到那些与当前这批区间完全不重叠的、更靠后的区间**，从而让后续的决策更轻松。

# 跳跃区间
下次你写出一个循环时，可以审视一下：我的循环体是不是太“臃肿”了？我是不是在里面做了太多“判断结论”的事情？我能不能让循环只负责“计算状态”，然后在循环结束后，用一个简单的 return 语句来得出结论？
在简洁架构（风格2）中，if i <= max_right: 这个判断起到了关键作用。

- **把它看作一个“卫语句 (Guard Clause)”**：
    
    - 你的代码是 if i > max_right: return False。这是一个“失败就立即退出”的信号。
        
    - 简洁架构的代码是 if i <= max_right: { ... }。这是一个“**只有在合法状态下，才执行核心逻辑**”的信号。
        
    - 这两种写法在逻辑上是等价的，但后者的心态是：“我只关心那些我能走到的 i，对于走不到的，我直接忽略（通过 else: break）”。这让 if 块内部的逻辑更纯粹——只处理有效情况。
        
- **把它看作循环的“不变式 (Invariant)”**：
    
    - “不变式”是一个在循环过程中需要始终保持为真的条件。
        
    - 在这里，i <= max_right 就是我们希望维持的“健康状态”。只要这个状态为真，我们的探索就可以继续。一旦它为假，探索就失去了意义。
        
    - 所以，把这个核心条件放在循环体的最外层，能让代码的意图更清晰：**“在我的探索还有意义的前提下（i <= max_right），我才去更新我的探索边界（max_right = max(...)）。**
