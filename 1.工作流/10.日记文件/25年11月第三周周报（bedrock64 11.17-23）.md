# 优化番茄钟：
### **一个完整的“满分日志”模板示例**

假设你正在攻克一道算法难题：


| 模块       | 你的填写内容                                                               | 分析师点评                                         |
| -------- | -------------------------------------------------------------------- | --------------------------------------------- |
| **任务启动** | 数据结构-DFS难题                                                           | 明确任务范围                                        |
| **第一步**  | **不写代码，先在草稿纸上画出递归树的前三层。**                                            | **极佳！** 将思维外化为物理动作，降低启动门槛。                    |
| **专注度**  | [4]                                                                  | 诚实的评分                                         |
| **状态记录** | 中间卡住了10分钟，**产生“我很笨”的念头（内部干扰），想放弃去喝水。深呼吸后，强制把自己拉回来，写下了当前卡住的点（应对措施）。** | **满分记录！** 捕捉到了情绪触发点，并记录了成功的干预措施。这对训练元认知极有价值。  |
| **结束总结** | 题没做出来。**但是理清了递归的边界条件。只差“回溯”的状态重置没想通。下次直接看这部分题解。**                    | **完美的钩子。** 承认进度（没做完），肯定收获（边界条件），明确下一步（看回溯题解）。 |

### **怎么执行优化？**

你不需要每一条都写这么详细（那样太累了）。  
**建议策略：**

1. **对于5分的番茄钟**：简单记录即可，保持流畅。
    
2. **对于3分及以下的番茄钟**：**必须**使用“诊断化”写法。多花1分钟，深挖为什么低分（是环境？情绪？还是任务难度？）。
    
3. **对于一天的最后一个番茄钟**：**必须**使用“钩子化”写法。为明天的启动铺路。
